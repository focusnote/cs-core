# 分析树

## 分析树是推导的图形化表示

分析树（Parse Tree），又称推导树（Derivation Tree），是上下文无关文法推导过程的图形化表示。

### 定义

对于文法 $G=(V_T, V_N, P, S)$，分析树满足：

1. **根节点**：标记为开始符号 $S$
2. **内部节点**：标记为非终结符 $A \in V_N$
3. **叶节点**：标记为终结符 $a \in V_T$ 或 $\varepsilon$
4. **产生式规则**：若内部节点 $A$ 的子节点从左到右为 $X_1, X_2, ..., X_n$，则 $A \rightarrow X_1X_2...X_n \in P$

### 分析树与推导的关系

- 每个推导对应一棵分析树
- 最左推导和最右推导对应相同的分析树（无二义性文法）
- 分析树忽略推导顺序，只关注结构

### 示例

文法：$E \rightarrow E+T \mid T$，$T \rightarrow T*F \mid F$，$F \rightarrow (E) \mid id$

表达式 `id + id * id` 的分析树：

```
        E
      / | \
     E  +  T
     |    /|\
     T   T * F
     |   |   |
     F   F   id
     |   |
    id  id
```

## （句型的）短语

### 短语（Phrase）

若 $S \stackrel{*}{\Rightarrow} \alpha A \beta \stackrel{*}{\Rightarrow} \alpha \gamma \beta$，则称 $\gamma$ 是句型 $\alpha\gamma\beta$ 相对于 $A$ 的**短语**。

**直观理解**：短语是句型中某棵子树的所有叶节点组成的符号串。

### 直接短语（Direct Phrase）

若 $S \stackrel{*}{\Rightarrow} \alpha A \beta \Rightarrow \alpha \gamma \beta$（只用一步推导），则称 $\gamma$ 是句型 $\alpha\gamma\beta$ 相对于产生式 $A \rightarrow \gamma$ 的**直接短语**。

**直观理解**：直接短语是句型中某棵简单子树（高度为1）的所有叶节点。

| 特性 | 短语 | 直接短语 |
|------|------|----------|
| 推导步数 | 多步 | 一步 |
| 子树高度 | 任意 | 高度为1 |

### 句柄（Handle）

**句柄**是句型的最左直接短语，是自底向上分析中归约操作的目标。

### 示例分析

句型 `id + id * id` 的短语分析：

| 短语 | 相对于非终结符 | 是否为直接短语 | 是否为句柄 |
|------|---------------|---------------|-----------|
| 第一个 `id` | $F, T, E$ | 是 | **是** |
| 第二个 `id` | $F, T$ | 是 | 否 |
| 第三个 `id` | $F$ | 是 | 否 |
| `id * id` | $T$ | 否 | 否 |
| `id + id * id` | $E$ | 否 | 否 |

---

## 子树与短语的关系

- **子树的叶节点**：构成一个短语
- **简单子树**（高度为1）：构成直接短语
- **最左简单子树**：构成句柄

---

## 二义性文法

### 定义

若存在一个句型有两棵不同的分析树，则该文法是**二义性文法**。

### 消除二义性

通过改写文法或添加优先级/结合性规则消除二义性。

---

## 语法树与分析树

### 语法树（Syntax Tree）

语法树是分析树的压缩形式，去除了中间的非终结符节点，只保留运算符和操作数。

### 区别

| 特性 | 分析树 | 语法树 |
|------|--------|--------|
| 节点 | 所有符号 | 仅运算符和操作数 |
| 结构 | 完整推导 | 压缩结构 |
| 用途 | 语法分析 | 语义分析、代码生成 |

### 示例

`id + id * id` 的语法树：

```
      +
     / \
    id  *
       / \
      id  id
```

---

## 分析树的应用

1. **语法分析**：验证输入串是否符合文法
2. **语义分析**：作为属性计算的基础结构
3. **代码生成**：遍历分析树生成中间代码或目标代码
4. **编译器优化**：基于分析树进行优化变换
